# 梅花易數與系統優化實作計畫 (Plum Blossom & Fixes)

## 問題修正 (Bug Fixes & UI Polishing)
1. **卦象資料自動化 (Data Reliability Fix)**
   - **原因**：目前手動更新的 JSON 或某些資料中，有時會漏掉 `hex.lower`（下卦）屬性，導致「畫卦天才」判斷失敗。
   - **解法**：在 `app.js` 初始化時，根據標準易經八卦命名法則自動運算 `hex.upper` 與 `hex.lower`（例如：「水雷屯」取第一第二字，「乾為天」取第三字）。如此不論外部 JSON 怎麼改，畫卦天才都能 100% 運作正確！
2. **「確認提交」按鈕隱藏 (Game 3 UI Fix)**
   - 在「畫卦天才」點擊送出後，立即以 `btnSubmitDraw.style.display = 'none'` 隱藏按鈕，並於進入下一題時再度顯示。
3. **滾動條體驗優化 (Scroll Behavior Optimization)**
   - 點開對話框時：強制將 `modal-content` 的 `scrollTop` 設為 0。
   - 切換頁籤（首頁、卦象學習、梅花易數）時：強制 `window.scrollTo(0, 0)`。
   - 網頁右下角新增一個 `↑` (回到最上面) 的浮動按鈕（Floating Action Button），點擊即可平滑捲動至頂部。

## 梅花易數模組 (Plum Blossom Divination Module)

### 1. 介面架構
在主控台新增第三個功能按鈕：「🌸 梅花易數」。
介面將分為三大區塊：
- **介紹區**：說明梅花易數的由來、核心（體用生剋、先天八卦數：乾1、兌2、離3、震4、巽5、坎6、艮7、坤8）。
- **起卦控制台**：
  - **時間起卦**：提供年(1-12)、月(1-12)、日(1-31)、時(1-12) 選單。
  - **數字起卦**：提供輸入 2 組數字。
  - 「開始排卦」按鈕。
- **結果解析區**：將產生排卦結果與吉凶斷言。

### 2. 起卦與五行運算邏輯
利用 Javascript 即時運算，不需依賴後端：
- **八卦對應五行**：
  - 乾、兌 ＝ 金
  - 震、巽 ＝ 木
  - 坎 ＝ 水
  - 離 ＝ 火
  - 坤、艮 ＝ 土
- **八卦建構**：
  - **上卦** = 數字 1 (或 年+月+日) `mod` 8
  - **下卦** = 數字 2 (或 年+月+日+時) `mod` 8
  - **動爻** = 總和 `mod` 6
- **動態尋卦與變更**：
  - 從 `trigramsData` 查出本卦的上下卦，再從 `activeHexagramData` 抓取對應的 64 卦實體。
  - 根據「動爻」位置：
    - 判斷「體卦 (Body)」與「用卦 (Use)」。動爻在下卦，下卦為用，上卦為體；動爻在上卦則反之。
    - 推算「互卦」：取本卦的 2,3,4 爻為下互卦，3,4,5 爻為上互卦。
    - 推算「變卦」：將該動爻陰陽顛倒，取得新的變卦（體卦不變，新用卦產生）。

### 3. 吉凶斷言邏輯 (生剋判斷)
自動對比「體卦的五行」與「用卦的五行」：
- **體生用**：洩氣、損耗。
- **用生體**：得生、大吉、有助力。
- **用剋體**：凶險、有阻礙。
- **體剋用**：偏吉、需靠努力克服困難方可成功。
- **體用比和**：大吉、平順同心。

此模組將為使用者提供極高互動性的解卦體驗。
